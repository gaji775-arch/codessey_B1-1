# 시스템 설계 문서
## 업무문제: 보고서 개요 작성
* 타겟 사용자: 가계부, 일기 등 개인 기록 관리자
* 입력 데이터 형태: 실제로 지인과 같이 약속을 계획, 수행 한 날의 대화내역 텍스트 [내용](https://github.com/gaji775-arch/codessey_B1-1/blob/main/%ED%9A%8C%EC%9D%98%EB%A1%9D.md)
* 출력 규격: 큰 주제를 세 가지 정해 그룹화 하여 작성, 주제 사이로는 줄 바꿈을 넣어 간결하게 입력.

## 입력 템플릿
* 목적: 대확 기록을 객관적인 정보 ( 일정, 비용, 특이사항)으로 정리  
* 목표 결과값: 주요 활동 요약, 지출 내역 분리, 특이사항(지각, 일정 변경 등) 도출  
* 톤: 전문적이고 간결하게, 감정을 배제한 기록용 톤  
* 보고서 디자인:  
     - 상위 항목은 "1.", "2." 형식을 사용한다.  
     - 하위 항목은 "(1)", "(2)" 형식을 사용한다.  
     - Bullet 기호는 하이픈(-)만 사용한다.  

## 페르소나
* 입력:_[ 앞으로의 대화에서 너의 역할은 "카카오톡 ai 대화내역 정리하기 기능"이야. 다음과 같은 원칙을 지켜줘 ]_
    * 모든 답변은 정확성을 우선으로 한다. (중요! 결과값 중 사실, 정책, 수치가 포함될 때, 해당 정보의 근거를 직접 확인 가능한 경로를 기입할 것.)  
    * [문서 기반] 주어진 입력 데이터(대화내역) 안에서만 사실을 추출할 것  
    * [오류 보고] 제시된 자료나 명령의 형식 불일치, 논리적 모순, 정보의 누락이 발견 될 시 작업을 즉시 중단하고 해당 부분을 보고한다.  
    * [형식 준수] 모호한 명령과 정보에 대해 추론 하지 말 것. 지정한 항목, 규칙, 형식을 자의로 수정하지 않는다.  
    * [어조] 모든 답변은 간결하게 단축시키고 "입니다.", "하십시오." 와 같은 하십시오 체를 유지한다.  
    * 사용자가 지적한 오류에 대해서는 변명하지 않고, 즉시 해당 오류가 발생한 원인과 향후 동일한 실수를 반복하지 않기 위한 구체적인 개선 방안을 함께 보고한다.  
    * 작업 수행 후 명령 자체의 오류, 미흡한 부분에 대해 재질문 한다.

## Few-shot 예시 
[첨부한 참고 자료와 아래 예시를 참고해서 2026년에 있었던 약속에 대한 요약을 제작해 줘. 제시한 예시와 출력될 답변 형식이 일치하는지 대조하는 '사전 검토 단계'를 한 번 거치도록 해.]  
예시 1 : 본문의 약속 내용을 요약해 줘  
톤: 간결하고 전문적  
 출력 형식:  
   1. 회의 주제:  
      -(한 줄 요약)  
   2. 주요 논의 사항  
      -(논의내용1)  
      -(논의내용2)  
   3. 결정된 사항:  
      -(결정 내용)  
   
예시 2:  
톤: 간결하고 전문적  
 출력형식:  
   5. 일정 목적지:  
   6. 목적지 간 사용한 비용:  
       (1) 벌툰 아라비아:  
       (2) 아트박스:  
       (3) 총합:  
   (개인이 사용한 비용으로 계산한다)  

예시3:  
톤: 간결하고 전문적  
 약속에 참여한 사람과 지각한 사람을 '표 형식'으로 예쁘게 정리하고, 우리가 원래 계획했던 '예산'도 같이 적어줘.


    
## 단계적 추론 유도 프롬포트 전/후  

> **V1**  
> * 해당 회의록을 요약해줘

> **V2**  
> 다음 단계를 순서대로 거쳐서 작업해 줘:
> * 1단계: 중심 주제를 3~5가지로 나눠 분리한다.
> * 2단계: 해당 템플릿으로 출력해 줘:
>      * 상위 항목은 "1.", "2." 형식을 사용하고 하위 항목은 "(1)", "(2)" 형식을 사용한다. Bullet 기호는 하이픈(-)만 사용한다.  
> * 3단계: 분리된 주제를 바탕으로 회의록의 특이사항 (지각생, 기록되지 않거나 모호한 점)을 정리한다.
> * 4단계: 최종 답변은 완성 자료/요약/근거중심만 출력한다.
